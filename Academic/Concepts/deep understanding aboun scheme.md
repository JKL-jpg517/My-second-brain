一切皆为表达式（Expression），一切都会算出一个值。
Scheme 的函数想象成一个“俄罗斯套娃”**或者**“算式替换”**的过程。判断一个结果是不是 return，只要看它是不是**“最后一步”**或者**“最外层”。

我们用 3 种典型场景来彻底分辨清楚：

### 场景一：算完就扔（不是 return）

当你在一个函数里，或者一个分支里，**平铺**了多个表达式时，只有最后一个才会被 return，前面的全都会被无情丢弃。

Scheme

```
(define (do-something x)
  (+ x 1)      ;; 第一步：算出了 x+1，但它不是最后一步。结果掉地上，扔了！
  (* x 2)      ;; 第二步：算出了 x*2，但它也不是最后一步。结果掉地上，扔了！
  (- x 3))     ;; 第三步：这是函数的最后一行（出口）。这个结果被 return 了！
```

**诊断：** 如果一个表达式的外面**既没有括号包裹它（比如把它当参数传给别人），它的下面又还有别的代码**，那它的结果100%会被丢弃。

_(这就是为什么你之前写的 `(append l ...)` 紧接着下一行又写了 `(my-filter ...)`，导致 `append` 的结果变成了废纸。)_

### 场景二：套娃传递（局部 return，作为中间材料）

当一个表达式被**包裹在另一个函数的括号里**时，它的结果会 return 给**包裹它的那个“上司”**，而不是直接 return 给最外面。

Scheme

```
(define (math-magic x)
  (+ 10 (* x 2))) 
```

**诊断：**

1. 计算机先看最里面：`(* x 2)`。它算出了一个值（比如 x=3，算出 6）。这个 6 会 return 吗？
    
2. 会，但它是 **return 给外层的 `+` 函数**，变成了 `(+ 10 6)`。
    
3. 然后外层的 `+` 执行，算出 16。
    
4. 因为 `+` 已经是这个函数的**最外层皮**（最后一步出口）了，所以这个 16 才真正被 return 到了函数外面。
    

### 场景三：完美出口（真正的 return）

在 Scheme 中，只要一个值出现在**逻辑流的最末端（尾巴上）**，它就会顺着管道直接滑出去，成为函数的最终 return 值。

在带有 `cond` 或 `if` 的代码里，出口非常清晰：

Scheme

```
(define (check-number x)
  (cond 
    ((> x 0) 'positive)    ;; 出口 1：如果条件成立，'positive 就是最后一步，直接 return！
    ((< x 0) 'negative)    ;; 出口 2：如果条件成立，'negative 就是最后一步，直接 return！
    (else    'zero)))      ;; 出口 3：兜底的最后一步，直接 return！
```

再看你之前写对的那段完美过滤代码：

Scheme

```
((pred (car s)) (cons (car s) (my-filter pred (cdr s))))
```

如果条件满足，解释器会执行后面这坨 `cons` 动作。 因为这坨 `cons` 外面没有别的东西包裹它了，它的下面也没有别的代码了（它已经是这个条件分支的最后一步）。所以，**这个 `cons` 拼接出来的全新列表，就直接成为了真正的 return 值！**

### 💡 终极判别法（请肌肉记忆）：

下次写代码时，拿你的笔尖指着你写的那行代码，问自己两个问题：

1. **它的外面还有没有别的一层函数在等它交差？** （比如外面有没有 `cons`、`+`、`list`？）如果有，它就是中间材料。
    
2. **它的下面还有没有别的代码要执行？** 如果有，它算出来的结果绝对会被当成垃圾扔掉！
    

如果外面没上司，下面没小弟，恭喜你，它就是真正的 **return（最终出口）**！

在 Linux 工具链中，这种“程序套程序”的用法非常经典，通常用于**测量、权限借调或环境改变**。你其实已经见过好几个类似的例子了：

| **结构**                 | **包装器程序 (裁判)** | **被包装的程序 (运动员)**(例子) | **实际作用**   |
| ---------------------- | -------------- | -------------------- | ---------- |
| **`time curl ...`**    | `time`         | `curl`               | 测量运行时间     |
| **`sudo tcpdump ...`** | `sudo`         | `tcpdump`            | 借调权限来运行程序  |
| **`strace ls`**        | `strace`       | `ls`                 | 监控程序的系统调用  |
| **`gdb ./app`**        | `gdb`          | `./app`              | 深入程序内部进行调试 |
- `sudo`：解决权限问题。
    
- `time`：解决性能分析（Profiling）问题。
    
- `strace`：解决底层调用排查问题。
    
- `bpftrace -c`：更高级的内核级追踪。

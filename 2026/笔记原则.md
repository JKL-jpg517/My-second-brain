 「高中化」笔记长什么样

  我看了你 vault 里几十篇笔记，模式很清晰：

  1. 转录思维，不是理解思维

  Strang 讲了什么 → 你记什么。顺序是 lecture 的顺序，不是概念本身的逻辑结构。L5 从 sum_digits 一路写到 Y combinator，因为 lecture 是这么讲的——但
  lecture 按这个顺序是为了控制节奏，不是为了帮你在三个月后检索。

  2. 收藏思维，不是取舍思维

  「这个可能会考」「那个可能有用」→ 全记下来。结果一篇笔记 150 行，核心概念只有 3
  个，被淹没在例子和证明中间。你不缺勤奋，你缺的是「这条我不记，因为我已经懂了」的判断。

  3. 被动引用，不是主动连接

  你用了 [[wikilink]]，但方式是「这两个东西有关系」——贴一个链接。高中思维是「归纳到错题本」，大学思维是「这个证明的 trick
  和另一个概念的结构是一样的」。

  怎么改：三条规则，现在就改

  规则 1：每篇笔记只能有一个「如果忘了这个，整篇就没意义」的核心句

  写任何笔记之前，先逼自己用一句话回答：这篇笔记究竟在讲什么？

  比如 4.1：
  - 高中写法：「正交向量、正交子空间、正交补、基本分解……」全写一篇
  - 大学写法：分三篇，每篇第一行就是那一个核心句
    - 4.1a：行空间和零空间正交，因为 Ax=0 的含义就是「A 的每一行与 x 内积为 0」
    - 4.1b：正交补意味着「包含了所有正交向量」——不仅是正交，而且是「对侧的全部」
    - 4.1c：任意 x 可唯一分解为 x_r + x_n，A 只对 x_r 起作用，x_n 被消灭

  如果你写不出这句核心句，说明你没理解。这篇笔记先别写，回去重新看。

  规则 2：笔记里必须有你自己的话，至少占 30%

  你现在的笔记 90% 是 lecture 内容的转写。一条硬指标：每记一个概念，后面跟一句你自己的话——类比、直觉、或者一个「为什么这不对」的反例。

  比如 L5d 树递归，原笔记只有 fib 代码和 count_partitions 代码。你自己的话应该是：

  ▎ 树递归的本质是：每个节点把自己拆成两个互不重叠的子问题，分别扔给两个子调用。它不怕重复计算——fib(3)
  ▎ 会被算很多次——所以树递归「对」但不「快」。后面 DP 要解决的就是这个重复计算。

  这句东西 lecture 不会替你写。但恰好是这句话让你三个月后翻开还能看懂。

  规则 3：每个概念笔记必须有一个「和什么有关」区块——不只是链接，是关系

  高中链接：[[knapsack]] ← 贴个名字，不知道为啥有关

  大学链接：
  - [[../Academic/Sources/数学/MIT 18.06 Linear Algebra/notes/4.2 Projections]]：投影到子空间 ← 4.1 的正交补是投影的几何基础
  - [[3.1 Spaces of Vectors]]：向量空间的定义 ← 4.1 是 3.1 的正交版本

  多了半句话，写清楚「为什么有关」。这半句话就是你脑子里概念网的网线。
